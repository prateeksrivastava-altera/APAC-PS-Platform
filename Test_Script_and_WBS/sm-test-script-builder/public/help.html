Getting Started

The Test Script Builder is an AI-powered tool that transforms your requirements documentation into structured, comprehensive test scripts. It uses a four-stage pipeline powered by Matcha AI to decompose documents, identify expected outcomes, build test flows, and compile a final test report.

Quick Start

  1. Upload your document
    Drag and drop (or click to browse) a .docx, .pdf, .txt, or .csv file containing your requirements or specifications.
  2. Select the content to test
    Choose which content to include — by headings, pages, or free-form text selection.
  3. Review and edit the prompt
    The assembled prompt is shown for your review. Toggle into edit mode to customise it before sending.
  4. Generate and download
    Click Generate Tests to run the AI pipeline. When complete, review the scenario cards and export to Excel or Azure DevOps format.
lightbulb Tip
You can navigate back to any completed step using the stepper bar at the top. Your selections and edits are preserved.

Uploading Documents

Step 1 lets you upload the requirements document that will be parsed and fed into the AI pipeline. The tool supports several common document formats.

Supported File Types

FormatExtensionParserNotes
Word Document .docx Mammoth.js Extracts headings (H1/H2) and body text. Best format for structured docs.
PDF Document .pdf PDF.js Extracts text by page. Heading detection uses font-size heuristics.
Plain Text .txt Built-in Treated as a single section. Best for short specs or notes.
CSV Spreadsheet .csv Built-in Treated as a single section with tabular content.

Upload Methods

  • Drag and drop — Drag a file from your desktop or file explorer directly onto the upload zone.
  • Click to browse — Click the upload zone or the Browse Files button to open the file picker.
warning Size Limit
Selected content must stay within the 400,000 character limit. Very large documents may require selecting less content in Step 2.
info Note
After a successful upload, the tool automatically advances to Step 2 (Content Selection). You can return to Step 1 to upload a different file at any time.

Content Selection

Step 2 offers three ways to select which content from your document is sent to the AI pipeline. A tab bar at the top lets you switch between modes.

Selection Modes

ModeIconDescriptionAvailable When
Headers checklist Select sections by their H1/H2 heading structure. Best for well-structured DOCX and PDF files. Document has detectable headings
Pages auto_stories Select individual pages or a range of pages. For PDFs these are real pages; for other formats they are approximate blocks. PDF files (always) or documents without headings
Text text_select Highlight and capture specific portions of text directly from the document. Always available for all file types
info Note
The tool automatically selects the best default mode based on your document. Greyed-out tabs indicate that mode is not available for this file type.

Headers Mode

  • Each section is displayed with a checkbox, its heading level (H1 or H2), and a character count.
  • Checking an H1 heading automatically checks all its child H2 sections beneath it.
  • Use Select All / Deselect All for quick batch operations.

Pages Mode

  • Use the page range selector at the top to quickly select a "from" and "to" range, then click Select Range.
  • Individual pages can also be toggled on/off with their checkboxes.
  • Each page shows its character count to help you stay within the limit.

Text Mode

  • The full document text is displayed in a scrollable panel.
  • Click and drag to highlight a region of text, then click Add Selection to capture it.
  • Multiple selections can be added — overlapping selections are merged automatically.
  • Each captured selection appears as a chip below the text area with a preview and a remove button.

Character Limit

Across all modes, the total selected content must remain below 400,000 characters. If you exceed this limit, the character counter turns red and the Continue button is disabled.

lightbulb Tip
For best results, select content that contains functional requirements rather than appendices, glossaries, or boilerplate. Switching between modes preserves your selections in each mode independently.

The AI Pipeline

Step 4 runs your assembled prompt through a four-stage AI pipeline. Each stage is a specialised Matcha AI mission that builds on the output of the previous stage.

Pipeline Stages

account_tree Identifying Test Areas
→
checklist Defining Expected Outcomes
→
route Creating Test Flows
→
summarize Compiling Test Report
StageMissionDescription
1 Identifying Test Areas Decomposes the document into testable functional areas and user-facing behaviours.
2 Defining Expected Outcomes Cross-references the original document to define expected results for each test area. Anti-hallucination rules ensure outputs are grounded in the source material.
3 Creating Test Flows Builds structured test scenarios with step-by-step flows, preconditions, and validation criteria.
4 Compiling Test Report Materialises the final output as a structured JSON report containing all test scenarios and their details.

Progress Indicators

  • Spinner icon — The currently running stage displays a spinning icon with a blue background.
  • Checkmark icon — Completed stages display a green checkmark.
  • Progress bar — A horizontal bar fills from 0% to 100% as each stage completes (25% per stage).
info Note
The pipeline typically takes 2 to 5 minutes depending on document size. Do not close the browser tab while it is running.

Reading Results

Step 5 displays the generated test scenarios as individual cards. Each card represents one test scenario from the AI output.

Scenario Card Structure

Example: SC-001 — Login with valid credentials
Scenario ID Unique identifier (e.g. SC-001)
Title Short description of the scenario
Priority High Medium Low

lightbulb Tip
The Notes column in each scenario card is editable. Click on any Notes cell to add your own annotations before exporting to Excel.

Downloading Excel Reports

The results toolbar offers two export formats: a standard Excel report and an Azure DevOps (AZDO) formatted export.

Standard Excel Export

Click the Export Excel button to download a .xlsx file with the following structure:

  • Each scenario is placed on its own worksheet, named by the scenario ID and title.
  • The 7-column format matches the on-screen table (Step, Action, Data, Expected Result, Precondition, Postcondition, Notes).
  • Headers are styled with a purple background and white text.
  • Alternating row shading for readability.
  • Auto-fitted column widths.

Azure DevOps Export

Click the Export AZDO button to generate a multi-sheet Excel file formatted for Azure DevOps import:

  • You will be prompted to enter an Area Path for the test plan.
  • Each scenario produces a worksheet with 9 columns including AZDO-specific fields (Work Item Type, Area Path).
  • The file can be imported directly into Azure DevOps Test Plans.

File Naming

Downloaded files follow the naming convention:
Test_Scenarios_YYYY-MM-DD_HH-MM.xlsx (standard) or
AZDO_Test_Scenarios_YYYY-MM-DD_HH-MM.xlsx (Azure DevOps).

info Note
Both export formats use the xlsx-js-style library to apply professional formatting including coloured headers, borders, and column sizing.

Raw JSON & Sent Text

The results view includes collapsible sections that let you inspect the raw data behind the generated test scenarios.

Raw JSON Output

Expand the Raw JSON toggle to see the complete JSON response from the AI pipeline. This is useful for:

  • Debugging when scenario cards are missing or incomplete.
  • Copying the raw output for use in other tools or workflows.
  • Verifying that the AI produced valid, well-structured output.

Sent Text

Expand the Sent Text toggle to review the exact prompt that was sent to the AI pipeline. This includes:

  • The system prompt (QA analyst instructions).
  • Your selected document sections.
  • Any manual edits you made in Step 3.
lightbulb Tip
If the results seem off, check the Sent Text to ensure the right sections were included and the prompt was not inadvertently modified.

Tips & Best Practices

Document Preparation

  1. Use structured headings. Documents with clear H1/H2 heading hierarchy produce the best section parsing results, especially in .docx format.
  2. Keep sections focused. Each section should cover a single feature or functional area. Avoid combining unrelated content under one heading.
  3. Prefer DOCX over PDF. Word documents preserve heading structure more reliably. PDF heading detection relies on font-size heuristics which can be imprecise.

Content Selection

  1. Be selective. Including too much content produces longer processing times and may dilute test quality. Focus on content containing functional requirements.
  2. Watch the character count. Stay well below the 400K limit for best results. Smaller, focused inputs generally produce higher quality tests.

Prompt Editing

  1. Review before generating. Always check the assembled prompt in Step 3 before running the pipeline. Ensure the system prompt and document content look correct.

Troubleshooting

Common issues and their solutions:

IssuePossible CauseSolution
File upload fails Unsupported file type or corrupted file Ensure the file is a .docx, .pdf, .txt, or .csv. Try re-saving the file.
No sections found after upload Document has no headings or uses non-standard formatting Add H1/H2 headings to your Word document. For PDFs, ensure text is selectable (not a scanned image).
Character limit exceeded Too much content selected Deselect content that is less relevant to testing, such as appendices or glossaries.
Pipeline stalls or times out Network issues or API overload Wait a moment and click Regenerate. If persistent, check your internet connection and try again.
Incomplete or missing scenarios Input too large or ambiguous for the AI Try with less, more focused content. Check the Raw JSON output to see what the AI returned.
Excel file is empty or malformed JSON output was not in the expected format View the Raw JSON to confirm valid output. Try regenerating the tests.
AZDO export missing area path Dialog was cancelled or left blank Click Export AZDO again and enter the area path when prompted (e.g. Project\Team\Sprint).
warning Important
If the pipeline fails repeatedly, check that the server is running (npm start) and that the MATCHA_API_KEY environment variable is correctly set in your .env file.
